Introduction

One of the main goals of the Data on the Web Best Practices ([DWBP]) is to facilitate interaction between publishers and consumers of data on the Web. A set of 35 Data on the Web Best Practices was created to cover different aspects related to data publishing and consumption, such as metadata, data licenses, data provenance, data quality, data versioning, data identifiers, data formats, data vocabularies, data access and APIs, data preservation, feedback, data enrichment, and republication. To show that the DWBP are implementable and broadly adopted and referenced, we adopted the following strategies:

  1. We have documented the existence of multiple evidences to show that a given best practice has been adopted by renewed organizations when publishing or consuming data on the Web. An evidence can be of different types, including: datasets; data portals; documents; references; guidelines; good practices.
  2. We have shown that each one of the proposed DWBP is implementable. Therefore, we have documented the existence of at least two implementations demonstrating the feasibility of the DWBP.

Evidence or implementation should cover one or more of the DWBP, regardless the strategy. During the implementation phase we looked for existing evidences as well as new implementations. This information was gathered as follows:

This report summarizes both surveys results and implementation details.

Methodology

We considered the following categories to collect evidence for the DWBP:

Datasets, Data Portals and Vocabularies:

This is the main category and concerns datasets, data portals and vocabularies available on the Web. The idea is to show that the DWBP were already considered by organizations responsible for publishing the data.
A dataset may be available on data catalogues and data portals may be implemented using technologies such as CKAN and Socrata, among others. In some cases, proprietary technologies were used to make the data available on the Web.
We also considered vocabularies as an evidence for the DWBP. It is important to note that publishing a vocabulary can be considered as specific case of publishing data on the Web.

Documents and References:

Besides the Datasets, Data Portals and Vocabularies we also considered other types of evidence that could show the relevance of a specific Best Practice.
This category includes web sites, web pages, blogs, published papers, APIs documentation, wikis, etc, that show the relevance of the DWBP.

Guidelines and Good Practices:

Finally, we also considered Guidelines and Good Practices, which contain links to public consultation that reference the DWBP.

Considering the categories above we used the following methodology to collect evidences for the DWBP:

  1. A standard email was sent to several organizations around the world in order to collect evidence.
  2. A review of the evidence was made in order to check which DWBP should have more evidence.
  3. The comments were reviewed among the evidence gathered.
  4. The Implementation Report was elaborated.

One of our main concerns when collecting evidences was to have evidences from renewed organizations as well as high profile datasets worldwide. Regarding that, we collected evidence from different types of organizations, in this document.

For some of the DWBP, such as Data Preservation, Real-time access, and Version history, was more difficult to find evidence on the form of datasets and data portals.

In the case of data Preservation, we received the comment saying that "archiving process has not been developed". About providing real-time access was said that "Generally the data is statistical in nature and goes through a slower collection and processing cycle".

On the other hand, most of the DWBP such as Metadata, Reuse vocabularies - preferably standardized ones, Data Formats, License, Data Identifiers, APIs, Feedback, and Data Enrichment.

In the case of Metadata we received a comment explaining that "A DCAT profile is provided in order to document the metadata". About Data Formats, was said for instance that "Currency values etc are provided together with units." and "Data is available as RDF in a number of serialisation formats." Considering the reuse of vocabularies, we received a comment saying that "A number of standard vocabularies are re-used, notably DCAT, RDF Data Cube, DCMI Terms." In the case of Data Identifiers was mentioned that "The dataset contents are provided as Linked Data, with (reasonably) persistent identifiers for geographical areas, time intervals and other statistical dimension values." Regarding License was said that "The license is provided in a machine readable form via the property and in human readable form "The license is provided in a machine readable form via the property and in human readable form."

Meeting the exit criteria

As noted in the DWBP charter, to move on to Proposed Recommendation, evidence will be adduced in order to demonstrate that each of the best practices has been recommended or adopted in at least two environments, such as data portals and formal policies. Evidence of implementation was gathered from existing datasets, which have already implemented the proposed best practices, as well as from national or sector-specific guidelines that reference the DWBP.
